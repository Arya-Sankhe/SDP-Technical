# Old Model Failure Notes

## `msft-gru-1m-recursive-forecast (1).ipynb`

This first GRU model failed because it tried to predict raw one-minute OHLC prices directly from a 500-candle history using only price columns and a simple MSE objective. Theoretically, this is a difficult setup because stock prices are non-stationary: the absolute price level changes over time, while the short-term signal we actually care about is the tiny return or movement between candles. A model trained on raw prices can learn the general price level but still miss the small minute-to-minute changes that determine forecasting quality. Technically, the results showed this clearly: the one-step close MAE was about `19.30` dollars, close RMSE was about `20.78`, and directional accuracy was only `0.485`, which is worse than a coin flip. The recursive 15-step forecast also stayed very poor, with close MAE around `18.65`. The model also suffered from recursive error accumulation: after each predicted candle was fed back as the next input, small mistakes became part of the future context, causing the forecast to drift farther away from reality. since we start training from an arbritarary price its hard for model to know if its expensive or not. small dataset problem.

## `msft-gru-1m-recursive-forecast (3).ipynb`

This version improved the representation by using transformed return-style candle features, volume/order-flow signals, multi-step training, scheduled sampling, and a larger GRU, but it still failed because the predictions collapsed toward a low-volatility average path. Theoretically, this is a common failure mode when using losses like Huber/MSE on noisy financial time series: because minute-level prices contain a lot of randomness and weak predictable signal, the safest way to minimize average error is to predict something close to the conditional mean instead of taking strong directional positions. Technically, this model was nearly tied with, or slightly worse than, a simple persistence baseline. Across two slices, one-step close MAE was about `0.287` vs persistence `0.284`, then `0.270` vs persistence `0.269`. The recursive step-15 forecast was also worse than persistence: `1.48` vs `1.39`, then `1.18` vs `1.13`. The strongest diagnostic was the predicted/actual close-return standard deviation ratio, only about `0.04` to `0.08`, meaning the model produced only a tiny fraction of the real market movement. In presentation terms, it learned to be smooth and safe, not useful.

## `msft-gru-1m-recursive-forecast (2).ipynb`

This version tried to fix collapse with a probabilistic decoder, Gaussian NLL loss, variance matching, lookback sweeps, imputation-aware features, and looser clipping, but it failed in the opposite direction: the model became too unstable and biased. Theoretically, adding uncertainty and variance objectives can help a model avoid flat predictions, but if the signal-to-noise ratio is low, those objectives may reward movement without teaching the model which movement is correct. That creates forecasts with more variance but not more information. Technically, the lookback sweep already showed the issue: the best candidate still had one-step close MAE around `0.953` compared with persistence around `0.262`. In the full walk-forward run, one-step close MAE was `0.637` and `0.830`, far worse than persistence at about `0.262`, and recursive step-15 MAE exploded to `3.93` and `17.08` compared with persistence near `1.29` and `1.11`. The standard deviation ratio improved to around `0.66` to `0.95`, so the model was no longer flat, but the extra movement was mostly wrong, producing large signed bias and severe recursive drift.

## `msft-gru-1m-recursive-forecast.ipynb`

This later anti-linear version was more careful: it used market-calendar sessionization, dropped high-fill sessions, skipped noisy open bars, predicted a candle geometry instead of raw OHLC, used probabilistic outputs, and extended the horizon to 30 steps. It still failed because the extra engineering reduced some obvious data problems but did not overcome the core forecasting problem: one-minute stock movement is extremely noisy, and a single-symbol GRU trained on roughly 60 days of MSFT data did not learn a stable edge over a naive baseline. Technically, the selected lookback was `96`, but the model's one-step close MAE averaged about `0.342` while persistence averaged about `0.230`, so the model was roughly 48 percent worse than simply assuming the next candle stays close to the previous one. Directional accuracy was also weak, around `0.48` to `0.50`, while persistence had stronger epsilon-based directional behavior in this evaluation. The recursive 30-step forecast failed even more clearly: model MAE was about `3.08` and `5.65`, compared with persistence around `1.16` and `1.26`. The standard deviation ratio of about `0.28` to `0.39` showed that even after anti-collapse fixes, the model still underrepresented real volatility and accumulated errors during rollout.

## Presentation Takeaway

The old models did not fail because GRUs are useless; they failed because the early setup asked a small single-stock model to extract a stable minute-level trading signal from noisy, non-stationary data. The progression is important: the first model failed from raw-price modeling and recursive drift, the next failed from volatility collapse, the next overcorrected into unstable forecasts, and the final old version still could not beat a simple persistence baseline. That failure path is what motivated the later improvements: better target design, stricter baselines, more realistic evaluation, and a clearer understanding that a model must beat simple market assumptions before it can be considered successful.
