# 5.2.3 Technical Analysis Implementation

## Market Data Processing
- Historical 1-minute OHLCV market data is retrieved first, then aligned into a clean intraday time series.
- Core market features are created from Open, High, Low, Close, Volume, Trade Count, and VWAP, along with return-based features.
- From the later notebooks, technical indicators such as RSI, MACD, SMA, EMA, ATR, Bollinger Bands, OBV, and VWAP-based deviations are added.
- The full feature matrix is normalized and converted into rolling windows so each model receives a fixed-length history and predicts the next trend or future candles.

## Models Considered and Their Pipelines

### Model 1: Trend-Aware GRU (`v7.1.ipynb`)
**Description of how it is used**  
This is the first major forecasting model in the notebook sequence. It uses a GRU-based sequence-to-sequence architecture with attention and adds trend-aware inputs such as multi-timeframe EMA slope and momentum. Its purpose is to learn short-term directional movement from recent price behavior.

**Implementation Pipeline**  
1. Retrieve OHLCV data and build base return and momentum features.  
2. Add trend-aware signals such as EMA slope and recent momentum.  
3. Create rolling input windows and feed them into the attention-based GRU forecaster.  
4. Predict future OHLC movement and a probabilistic price path.

### Model 2: Technical-Indicator GRU (`v9.1.ipynb`)
**Description of how it is used**  
This model keeps the GRU forecasting backbone but improves the input representation by adding a dedicated technical-indicator engine. It uses RSI, MACD, SMA, EMA, ATR, Bollinger Bands, OBV, and VWAP-style features to give the network stronger trading context. This makes the model more informative than the earlier trend-only version.

**Implementation Pipeline**  
1. Compute technical indicators from the raw OHLCV stream.  
2. Concatenate technical indicators with the core price and return features.  
3. Convert the enriched features into rolling windows for training and testing.  
4. Use the GRU model to output future trend direction or price movement.

### Model 3: Regime-Aware GRU (`v9.2.ipynb`)
**Description of how it is used**  
This model extends the GRU pipeline by adding market-regime detection. It classifies the market into states such as normal, elevated, or crisis using turbulence-style logic, then adjusts inference behavior accordingly. This helps the model react differently under stable and unstable market conditions.

**Implementation Pipeline**  
1. Build the GRU forecasting pipeline on the prepared intraday feature windows.  
2. Detect the current market regime using regime indicators and turbulence measures.  
3. Adjust sampling temperature and inference behavior based on the detected regime.  
4. Produce a regime-aware prediction with a corresponding confidence level.

### Model 4: FT-iTransformer / Time-Frequency Forecast Model (`v9.4.ipynb`)
**Description of how it is used**  
This is the most advanced single forecasting model in the notebook series. It combines the original sequence-learning pipeline with multi-scale STFT-based frequency features, allowing the model to learn both temporal structure and repeating market cycles. It is stronger than the plain GRU models for complex intraday behavior.

**Implementation Pipeline**  
1. Prepare the same rolling windows used by the GRU-based models.  
2. Extract multi-scale frequency representations using STFT-based blocks.  
3. Combine time-domain and frequency-domain information inside the FT-iTransformer architecture.  
4. Generate the final trend or price forecast together with uncertainty estimates.

### Model 5: Final Fusion Ensemble (Current Model)
**Description of how it is used**  
The final model used in the system is a fusion ensemble built from the strongest earlier models. It takes the outputs of the Trend-Aware GRU, Technical-Indicator GRU, Regime-Aware GRU, and FT-iTransformer model, then averages their results into a single final prediction. This improves stability and reduces reliance on any one model.

**Implementation Pipeline**  
1. Run all selected models on the same latest input window.  
2. Convert their outputs into a common prediction format.  
3. Average the predicted direction scores or predicted prices across models.  
4. Return one fused prediction and one fused confidence score.

## Module Output Format
- **Predicted trend direction or price:** Each model outputs either a direction label such as `Up`, `Down`, or `Neutral`, or a predicted future price/path.
- **Confidence or trend-strength metric:** Each model also outputs a confidence score or trend-strength value that reflects how strong the signal is.
- **Output structure for the fusion engine:** All model outputs are converted into the same structure before averaging.

```json
{
  "model_name": "FT-iTransformer",
  "predicted_direction": "Up",
  "predicted_price": 425.80,
  "confidence": 0.81,
  "trend_strength": 0.76
}
```

The fusion engine reads this format from the contributing models, averages the relevant values, and produces the final system prediction.
