# v9.6 Technical Learning Summary

`v9.6.ipynb` is best understood as `v8.5` plus a separate reinforcement-learning trading layer. The price forecaster itself is still the same probabilistic seq2seq attention GRU with ensemble trend selection and a strictly causal rolling backtest. The new part is a PPO policy that consumes rolling forecast paths as state and learns trading actions with a Differential Sharpe Ratio style reward.

## What changed vs v8.5

The forecasting side is basically unchanged:

- same `MSFT` 1-minute data pipeline
- same `10` core engineered features
- same probabilistic GRU forecaster
- same autoregressive path sampling
- same trend-matched ensemble selection
- same strictly causal rolling walk-forward backtest

What v9.6 adds is a post-forecast decision layer:

1. run the causal rolling forecaster across one full session
2. use each rolling prediction path as part of an RL state
3. add current market indicators and portfolio state
4. feed that state into a PPO actor-critic policy
5. score actions with a streaming risk-adjusted reward

So this version does **not** replace the forecaster. It keeps forecasting and trading as two separate stages.

- decoder: autoregressive `GRUCell`
- attention: dot-product attention over encoder states
- outputs: `mu` and `log_sigma` for each OHLC return

So the model still predicts a Gaussian distribution over future returns rather than a single point estimate.

- `BATCH_SIZE = 256`

- gradient clipping at `1.0`

Loss:

`total = weighted_nll + 0.3 * range_loss + 0.5 * volatility_match_loss + 0.1 * directional_penalty`

The notebook samples multiple future paths, computes a close-price trend slope for each one, and keeps the path whose slope best matches the recent historical slope.

## RL decision layer

This is the actual new technique in v9.6.

After the rolling backtest finishes, the notebook builds an RL dataset from the rolling outputs:

- `rl_rows = 390`
- `prediction_tensor_shape = (390, 50, 4)`

Each RL observation is:

- flattened forecast path: `50 * 4 = 200` values
- `10` market features
- `4` portfolio features
- `1` regime indicator

So the saved environment state dimension is:

- `215`

The `10` market features are:

- RSI-14
- MACD histogram
- Bollinger-band position
- ATR-14 percentage
- 5-bar price momentum
- 20-bar price momentum
- VWAP-20 deviation
- OBV slope
- direction
- relative volume

Portfolio state contains:

- normalized cash balance
- normalized share count
- normalized portfolio value
- current position percentage

### Action space

The action is one continuous scalar in `[-1, 1]`.

It represents target portfolio exposure:

- negative = short
- positive = long
- magnitude = fraction of max allowed position

The environment converts that into a target dollar position, trades toward it, and charges:

- `transaction_cost = 0.001`
- `max_position = 1.0`

Trades smaller than `$10` are ignored.

### Reward

Reward is based on a streaming Differential Sharpe Ratio style update:

- maintain exponentially updated moments `A` and `B`
- update them with `dsr_eta = 0.1`
- reward each step by a local risk-adjusted return improvement term

So the RL target is not raw PnL. It is a risk-adjusted objective.

### Regime-aware sizing

The environment also includes a market-regime detector:

- turbulence index from recent returns
- ATR percentile filter
- `lookback = 60`

Regimes:

- `NORMAL`
- `ELEVATED`
- `CRISIS`

Position multipliers:

- normal: `1.0`
- elevated: `0.7`
- crisis: `0.3`

Regime indicator values in state:

- normal: `0.0`
- elevated: `0.5`
- crisis: `1.0`

So the PPO policy proposes an action, but the final exposure is scaled down automatically in stressed regimes.

### PPO implementation

The policy network is a simple MLP actor-critic, not a sequence model:

- two shared `Linear -> LayerNorm -> Tanh` blocks
- hidden size `256`
- actor mean head with `tanh`
- learned log-standard-deviation parameter
- scalar critic head

PPO hyperparameters:

- `ppo_lr = 3e-4`
- `ppo_gamma = 0.99`
- `ppo_gae_lambda = 0.95`
- `ppo_clip_epsilon = 0.2`
- `ppo_value_coef = 0.5`
- `ppo_entropy_coef = 0.01`
- `ppo_max_grad_norm = 0.5`
- `ppo_rollout_length = 512`
- `ppo_update_epochs = 4`
- `ppo_batch_size = 64`
- `rl_training_steps = 5000`

The notebook implements:

- generalized advantage estimation
- clipped PPO surrogate objective
- value loss
- entropy bonus
- gradient clipping

## Main takeaway

v9.6 is not a new forecasting model. It is:

- the same probabilistic GRU forecaster from v8.5
- the same ensemble trend-selection inference
- plus a new PPO trading layer that consumes strictly causal rolling forecast paths

If you were reimplementing it, the important pieces are:

- keep the forecasting model and trading policy separate
- generate causal rolling predictions first
- flatten the full predicted future path into the RL state
- combine that with market features, portfolio state, and a regime flag
- optimize a continuous position-sizing policy with PPO and a DSR-style reward

The main limitation of the saved notebook is that the RL layer is only scaffolded and sanity-checked, not actually trained or benchmarked end to end.
