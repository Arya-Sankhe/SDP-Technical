# 5.2.3 Technical Analysis Implementation

## Market Data Processing
- Historical OHLCV data is retrieved for the target stock and cleaned into a consistent time-indexed format.
- Standard technical indicators are then computed, including RSI, MACD, SMA, EMA, Bollinger Bands, ATR, OBV, and VWAP.
- The enriched feature set is normalized and converted into rolling windows so each model receives a fixed-length historical sequence.
- These windowed sequences are paired with future labels such as next-step price movement, future OHLC values, or trend direction.

## Models Considered and Their Pipelines

### Model 1: Technical Indicator-Based Model
**Description of how it is used**  
This model uses handcrafted technical indicators as direct trading signals. It interprets patterns such as RSI overbought/oversold levels, MACD crossovers, and SMA/EMA trend alignment to estimate short-term direction. It is simple, transparent, and useful as a strong baseline.

**Implementation Pipeline**  
1. Compute RSI, MACD, SMA, EMA, ATR, Bollinger Bands, OBV, and VWAP from OHLCV data.  
2. Convert indicator conditions into bullish, bearish, or neutral signals.  
3. Aggregate the signals into a trend score and confidence value.  
4. Output the indicator-based prediction to the fusion engine.

### Model 2: GRU
**Description of how it is used**  
The GRU model learns temporal dependencies from sequential market data. It uses rolling windows of OHLCV and technical features to model recent price behavior and forecast the next movement or future price path. It is effective because it keeps memory of recent market structure while remaining lighter than more complex transformers.

**Implementation Pipeline**  
1. Build fixed-length time-series windows from normalized OHLCV and technical features.  
2. Feed each window into the GRU encoder-decoder forecasting model.  
3. Generate a predicted trend direction or future price output from the hidden sequence representation.  
4. Pass the GRU prediction and confidence score to the fusion engine.

### Model 3: FT-iTransformer / Hybrid Time-Frequency Model
**Description of how it is used**  
This model extends sequence forecasting by combining time-domain learning with frequency-domain analysis. It captures both cross-feature relationships and repeating market cycles, which makes it stronger for complex patterns than a plain GRU. It is used as the advanced deep-learning model in the pipeline.

**Implementation Pipeline**  
1. Prepare the same rolling windows used for the GRU model.  
2. Extract time-frequency features using STFT or multi-scale frequency blocks.  
3. Process the sequence with the hybrid iTransformer-based architecture to predict price movement.  
4. Send the predicted value and confidence estimate to the fusion engine.

### Model 4: Fusion Ensemble Model
**Description of how it is used**  
This is the final model used in the system. It combines the outputs of the technical indicator model, the GRU model, and the FT-iTransformer model, then averages them to produce one final prediction. This improves robustness because the final output is not dependent on a single model alone.

**Implementation Pipeline**  
1. Collect prediction outputs from Model 1, Model 2, and Model 3.  
2. Normalize their scores into a common output format.  
3. Average the predicted prices or direction scores across models.  
4. Return the final ensemble prediction to the decision layer.

## Module Output Format
- **Predicted trend direction or price:** Each model outputs either `Up`, `Down`, or `Neutral`, and/or a predicted future price.
- **Confidence or trend-strength metric:** Each output includes a confidence score showing how strong or reliable the prediction is.
- **Output structure for the fusion engine:** A simple structure such as:

```json
{
  "model_name": "GRU",
  "predicted_direction": "Up",
  "predicted_price": 425.80,
  "confidence": 0.78,
  "trend_strength": 0.72
}
```

The fusion engine reads this same format from all models, averages the prediction values, and produces the final system output.
