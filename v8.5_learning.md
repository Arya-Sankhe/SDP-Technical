# v8.5 Technical Learning Summary

`v8.5rollingbacktest.ipynb` is a probabilistic intraday OHLC forecaster for `MSFT` 1-minute bars. The core model is still the v7.5 seq2seq attention GRU; v8.5 mainly adds a strictly causal rolling walk-forward backtest and frame generation on top of that training pipeline.

## What the model is actually doing

The notebook fetches `120` days of 1-minute bars from Alpaca, sessionizes them against the NYSE calendar, and drops sessions with too much missing data. In the saved run it kept `80 / 83` sessions, with `30,840` usable rows after session filtering. Missing OHLC/VWAP values are filled within each session, volume and trade count are zero-filled, and an imputation flag is preserved so contaminated targets can be excluded later.

Each bar is converted into return-style features relative to the previous close:

- `rOpen`, `rHigh`, `rLow`, `rClose`
- `logVolChange`, `logTradeCountChange`
- `vwapDelta`
- `rangeFrac`
- `orderFlowProxy`
- `tickPressure`

The prediction target is the next `50` steps of OHLC returns. Inputs are standardized using training-only statistics, but targets are not standardized. Windows are dropped if any target bar is imputed or falls inside the first `6` minutes after the open.

## Model architecture

The model is `Seq2SeqAttnGRU`:

- Encoder: `GRU(input_size=11, hidden_size=256, num_layers=2, dropout=0.20, batch_first=True)`
- Decoder: one `GRUCell` run autoregressively for `HORIZON=50`
- Attention: dot-product style attention over encoder states
- Heads: one head predicts `mu`, one predicts `log_sigma` for each OHLC return

So this is not a point forecaster. At every future step the network predicts a Gaussian distribution over returns, then sampling is used at inference time to generate full paths.

## Training recipe

Training uses:

- `AdamW`, learning rate `5e-4`, weight decay `1e-5`
- Batch size `256`
- Scheduled teacher forcing from `1.0` down with decay `0.95` each epoch
- `ReduceLROnPlateau(factor=0.5, patience=3, min_lr=1e-6)`
- Gradient clipping at `1.0`

The loss is:

`total = NLL + 0.3 * range_loss + 0.5 * volatility_match_loss + 0.1 * directional_penalty`

Important implementation details:

- The main term is Gaussian negative log likelihood on the OHLC return targets.
- Later forecast steps are weighted more heavily with `1 + (t / (H-1))^1.5`.
- `range_loss` pushes predicted candle range toward the true range.
- `volatility_match_loss` pushes predicted sigma toward realized target volatility.
- `directional_penalty` penalizes opposite-sign close-return predictions.

## Path generation and trend selection

Inference is the main technique change from the earlier versions. Instead of taking only the decoder mean, the notebook generates an ensemble of stochastic future paths:

- `ENSEMBLE_SIZE = 20`
- `SAMPLING_TEMPERATURE = 1.5`
- `TREND_LOOKBACK_BARS = 20`
- `STRONG_TREND_THRESHOLD = 0.002`
- `MIN_PREDICTED_VOL = 0.0001`

`generate_realistic()` samples recursively from `mu + epsilon * sigma`, where `sigma = exp(log_sigma) * temperature`. For the first `5` steps it can override sigma using recent realized volatility from the context window. After converting returns back to prices, each candle is forced to satisfy `High >= max(Open, Close)` and `Low <= min(Open, Close)`.

The ensemble is then filtered by trend continuation:

1. Compute the historical slope from the last `20` close prices in the context.
2. Generate `20` sampled future paths with different seeds.
3. Compute each path slope from predicted close returns.
4. If the historical trend is strong, reject paths moving in the opposite direction.
5. Select the path with minimum slope distance to the historical slope.

This is the notebook's main "smarter inference" idea: keep stochastic generation, but bias final selection toward momentum continuation instead of picking an arbitrary sample.

## v8.5 rolling backtest extension

The v8.5-specific addition is a bar-by-bar rolling backtester that keeps causality strict.

Key mechanics:

- Backtest date is auto-selected to the latest valid full session with future labels available.
- In the saved run the selected date was `2026-02-19`.
- The scaler is fit only on rows before the backtest date.
- Training windows are kept only if their full target horizon ends before the backtest date.
- At anchor time `t`, the model only sees `[t-window, ..., t-1]`.
- Predictions are generated for every minute from `09:30` to `16:00` with `ROLLING_STEP = 1`.

Temperature is also scheduled intraday:

- `09:30-10:15 -> 1.25`
- `10:15-14:00 -> 1.45`
- `14:00-16:00 -> 1.60`

The rolling engine logs each anchor, predicted path, actual path, first-step directional hit, and stepwise MAE. It also asserts:

- first predicted timestamp equals the anchor timestamp
- context strictly ends at `t-1`
- prediction count matches expected minute anchors

In the saved run it produced `390` predictions and saved `390` standalone frames.

## Observed results and caveat

On the full walk-forward run, the selected `64`-bar model produced:

- `slice_1` model close MAE: `0.3117` vs persistence `0.2183`
- `slice_2` model close MAE: `0.2961` vs persistence `0.2125`

Rolling-backtest summary:

- directional hit rate at `t+1`: `49.49%`
- path MAE at step 1: `$0.2444`
- path MAE at step 5: `$0.5265`
- path MAE at step 10: `$0.8640`
- path MAE at step 15: `$1.1194`
- trend correlation: `-0.0151`

So the notebook is best understood as a realistic stochastic path generator with causal evaluation, not as a one-step alpha model that already beats a persistence baseline. If you re-implement it, the important pieces are the probabilistic seq2seq decoder, volatility-aware sampling, trend-matched ensemble selection, and the strict no-future-data rolling evaluation.
